1.Отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory) можно с помощью команды git checkout:
  git checkout - name file - 
2.Чтобы вернуть файл из индекса в рабочий каталог, используйте команду git reset:
  git reset - name file - 
3.Чтобы отменить ошибочный commit, можно использовать команду git reset. Если вы хотите отменить последний коммит, вы можете выполнить:
  git reset HEAD~1 - где HEAD означает один коммит назад
4.Чтобы создать новую ветку и переключиться на нее, используйте команду git checkout с ключем -b:
  git checkout -b - name branch - 
5.Fast-forward merge происходит, когда все изменения, внесенные в целевую ветку после того, как была создана ветка для слияния, 
  могут быть просто перемещены в ветку для слияния без создания новых коммитов. Не fast-forward merge происходит, когда в ветках, 
  которые вы сливаете, есть конфликты и Git создает новый коммит, который объединяет обе ветки.

6.Конфликт слияния веток может возникнуть, если изменения в ветках, которые вы сливаете, конфликтуют между собой. Чтобы разрешить конфликт, 
  необходимо вручную решить, какие изменения следует сохранить, а какие отклонить. Git помечает конфликтующие участки в файлах с помощью 
  специальных маркеров и показывает вам изменения из каждой ветки. После того, как вы решите, какие изменения следует сохранить, вы должны 
  сделать новый коммит, чтобы завершить слияние.	
7.Да, ветку можно удалить с помощью команды git branch -d:
  git branch -d - name branch -
